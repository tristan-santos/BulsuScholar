Material Checking Seeder

Seed 50 pending SOE checking records from live student data.

This tool reads the current students, soeRequests, and soeDownloads collections, then creates pending soeDownloads records that match actual student snapshots and scholarship data for the Materials Checking section.

Uses real student snapshots Prefers approved SOE requests Writes to soeDownloads

Usage

Run npm run dev, open /seed.html, then sign in as an admin if your Firestore rules require it.

Seeder Scope

New records are marked with seed metadata so you can delete the latest batch or clear all rows created by this page without touching manual data.

Students Loaded 0 Live student records pulled from Firestore.
Eligible Pairs 0 Student and scholarship pairs suitable for SOE checking seed data.
Request-Backed 0 Pairs that already have a matching SOE request to reuse.
Existing Checking Rows 0 Current documents already inside soeDownloads.

Seed Controls

Preview the generated rows first, then seed them into Materials Checking. If the database has fewer than 50 unique candidates, the tool reuses real students and scholarships with unique SOE request numbers to still reach the target.
